## Build a Blog Workspace

In this guide, you'll create a fully functional blog workspace with posts and tags, complete with table operations and type-safe queries.

<Steps>
  <Step title="Install Epicenter">
    First, install the Epicenter workspace package:

    ```bash
    bun add @epicenter/workspace
    ```

    <Note>
      Epicenter uses Bun for package management and runtime. If you don't have Bun installed, see the [Installation](/installation) guide.
    </Note>
  </Step>

  <Step title="Define Your Workspace">
    Create a new file `blog-workspace.ts` and define your workspace schema:

    ```typescript blog-workspace.ts
    import { defineWorkspace, defineTable } from '@epicenter/workspace';
    import { type } from 'arktype';

    // Define the posts table with version 1 schema
    const postsTable = defineTable(
      type({
        id: 'string',
        title: 'string',
        content: 'string',
        published: 'boolean',
        views: 'number',
        _v: '1' // Version discriminant (required)
      })
    );

    // Define the tags table
    const tagsTable = defineTable(
      type({
        id: 'string',
        name: 'string',
        _v: '1'
      })
    );

    // Compose the workspace definition
    export const blogWorkspace = defineWorkspace({
      id: 'blog',
      tables: {
        posts: postsTable,
        tags: tagsTable
      }
    });
    ```

    <Tip>
      The `_v` field is a version discriminant that enables schema migrations. Even single-version tables require it.
    </Tip>
  </Step>

  <Step title="Create a Workspace Client">
    Initialize a workspace client to interact with your data:

    ```typescript blog-client.ts
    import { createWorkspace } from '@epicenter/workspace';
    import { blogWorkspace } from './blog-workspace';

    // Create the client
    const client = createWorkspace(blogWorkspace);

    // The client is now ready to use!
    console.log('Workspace ID:', client.id); // 'blog'
    ```

    <Note>
      This creates an in-memory workspace. Data persists only while the process runs. Add persistence in the next step.
    </Note>
  </Step>

  <Step title="Add Extensions (Optional)">
    Extend your workspace with persistence, SQLite queries, or sync:

    <CodeGroup>
      ```typescript Persistence Only
      import { createWorkspace } from '@epicenter/workspace';
      import { persistence } from '@epicenter/workspace/extensions';
      import { blogWorkspace } from './blog-workspace';

      const client = createWorkspace(blogWorkspace)
        .withExtension('persistence', persistence);

      // Data now persists to filesystem (.epicenter/persistence/blog.yjs)
      ```

      ```typescript SQLite + Persistence
      import { createWorkspace } from '@epicenter/workspace';
      import { persistence, sqlite } from '@epicenter/workspace/extensions';
      import { blogWorkspace } from './blog-workspace';

      const client = createWorkspace(blogWorkspace)
        .withExtension('persistence', persistence)
        .withExtension('sqlite', sqlite);

      // Enable SQL queries via client.extensions.sqlite
      ```

      ```typescript Full Stack (SQLite + Markdown + Sync)
      import { createWorkspace } from '@epicenter/workspace';
      import { persistence, sqlite, markdown, sync } from '@epicenter/workspace/extensions';
      import { blogWorkspace } from './blog-workspace';

      const client = createWorkspace(blogWorkspace)
        .withExtension('persistence', persistence)
        .withExtension('sqlite', sqlite)
        .withExtension('markdown', markdown)
        .withExtension('sync', sync({
          url: 'ws://localhost:3913/rooms/{id}'
        }));

      // Full-featured: SQL queries, markdown files, and real-time sync
      ```
    </CodeGroup>
  </Step>

  <Step title="Use Table Operations">
    Perform CRUD operations on your tables:

    ```typescript table-operations.ts
    import { generateId } from '@epicenter/workspace';

    // Create a post
    const postId = generateId();
    client.tables.posts.set({
      id: postId,
      title: 'Hello Epicenter',
      content: 'My first post using Epicenter!',
      published: false,
      views: 0,
      _v: 1
    });

    // Read a post
    const result = client.tables.posts.get(postId);
    if (result.status === 'valid') {
      console.log('Post:', result.row.title);
    }

    // Update a post
    client.tables.posts.update(postId, { views: 42 });

    // Get all posts
    const allPosts = client.tables.posts.getAll();
    console.log(`Total posts: ${allPosts.length}`);

    // Filter posts
    const published = allPosts.filter(r => 
      r.status === 'valid' && r.row.published
    );

    // Delete a post
    client.tables.posts.delete(postId);
    ```

    <Tip>
      Use `generateId()` to create unique IDs. It generates URL-safe nanoid strings.
    </Tip>
  </Step>

  <Step title="Query with SQLite (Optional)">
    If you added the SQLite extension, use Drizzle ORM for complex queries:

    ```typescript sqlite-queries.ts
    import { eq, desc, and } from '@epicenter/workspace';

    // Get the sqlite extension
    const { posts } = client.extensions.sqlite;

    // Query published posts ordered by views
    const topPosts = await posts
      .select()
      .where(eq(posts.published, true))
      .orderBy(desc(posts.views))
      .limit(10);

    console.log('Top 10 posts:', topPosts);

    // Complex query with multiple conditions
    const recentPopular = await posts
      .select()
      .where(
        and(
          eq(posts.published, true),
          posts.views.gte(100)
        )
      );
    ```

    <Note>
      The SQLite extension automatically syncs with your YJS document. Write to YJS, query from SQLite.
    </Note>
  </Step>

  <Step title="Observe Changes">
    React to real-time changes with observers:

    ```typescript observers.ts
    // Watch for changes to posts
    const unsubscribe = client.tables.posts.observe((changedIds) => {
      console.log('Changed posts:', changedIds);
      
      for (const id of changedIds) {
        const result = client.tables.posts.get(id);
        if (result.status === 'valid') {
          console.log('Updated post:', result.row.title);
        }
      }
    });

    // Make some changes - observer will fire
    client.tables.posts.set({
      id: generateId(),
      title: 'New Post',
      content: 'This triggers the observer',
      published: true,
      views: 0,
      _v: 1
    });

    // Clean up when done
    unsubscribe();
    ```

    <Tip>
      Observers fire for both local and remote changes, making them perfect for building reactive UIs.
    </Tip>
  </Step>

  <Step title="Cleanup">
    Always destroy the client when you're done:

    ```typescript cleanup.ts
    // Manual cleanup
    await client.destroy();

    // Or use automatic cleanup with 'await using'
    {
      await using client = createWorkspace(blogWorkspace)
        .withExtension('persistence', persistence);
      
      // Use the client...
      client.tables.posts.set({ /* ... */ });
      
      // Automatically destroyed when block exits
    }
    ```
  </Step>
</Steps>

## Complete Example

Here's the full code in one file:

<CodeGroup>
```typescript Complete Example
import { 
  createWorkspace, 
  defineWorkspace, 
  defineTable,
  generateId 
} from '@epicenter/workspace';
import { persistence, sqlite } from '@epicenter/workspace/extensions';
import { type } from 'arktype';
import { eq, desc } from '@epicenter/workspace';

// 1. Define workspace
const postsTable = defineTable(
  type({
    id: 'string',
    title: 'string',
    content: 'string',
    published: 'boolean',
    views: 'number',
    _v: '1'
  })
);

const blogWorkspace = defineWorkspace({
  id: 'blog',
  tables: { posts: postsTable }
});

// 2. Create client with extensions
await using client = createWorkspace(blogWorkspace)
  .withExtension('persistence', persistence)
  .withExtension('sqlite', sqlite);

// 3. Create a post
const id = generateId();
client.tables.posts.set({
  id,
  title: 'Hello Epicenter',
  content: 'My first post!',
  published: true,
  views: 0,
  _v: 1
});

// 4. Query with SQL
const { posts } = client.extensions.sqlite;
const published = await posts
  .select()
  .where(eq(posts.published, true))
  .orderBy(desc(posts.views));

console.log('Published posts:', published);

// 5. Automatic cleanup on block exit
```

```typescript With Sync
import { 
  createWorkspace, 
  defineWorkspace, 
  defineTable,
  generateId 
} from '@epicenter/workspace';
import { persistence, sync } from '@epicenter/workspace/extensions';
import { type } from 'arktype';

// Define workspace
const postsTable = defineTable(
  type({
    id: 'string',
    title: 'string',
    published: 'boolean',
    _v: '1'
  })
);

const blogWorkspace = defineWorkspace({
  id: 'blog',
  tables: { posts: postsTable }
});

// Create client with real-time sync
await using client = createWorkspace(blogWorkspace)
  .withExtension('persistence', persistence)
  .withExtension('sync', sync({
    url: 'ws://localhost:3913/rooms/{id}'
  }));

// Changes sync automatically across all connected clients
client.tables.posts.set({
  id: generateId(),
  title: 'Synced Post',
  published: true,
  _v: 1
});

// This change will appear on all connected devices!
```
</CodeGroup>

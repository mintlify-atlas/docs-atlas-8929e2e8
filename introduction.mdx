## Welcome to Epicenter

Epicenter is a unified workspace architecture built on YJS for real-time collaboration with optional persistence and query layers. It enables you to build local-first, offline-capable applications with conflict-free data synchronization.

## Core Philosophy

### YJS Document as Source of Truth

Epicenter uses YJS documents as the single source of truth for all data. YJS provides:

- **CRDT-based conflict-free merging** - Multiple users can edit simultaneously without conflicts
- **Real-time collaborative editing** - Changes sync instantly across all connected clients
- **Built-in undo/redo** - Full operation history at no extra cost
- **Efficient binary encoding** - Minimal network overhead and storage

### Unified Extensions for Querying and Persistence

Extensions are capabilities that can mirror YJS data:

- **SQLite Extension** - Enables SQL queries via Drizzle ORM for complex filtering and aggregations
- **Markdown Extension** - Persists data as human-readable markdown files
- **Sync Extension** - Real-time multi-device synchronization via WebSocket
- **Custom Extensions** - Build your own (vector search, full-text search, etc.)

Extensions auto-sync bidirectionally with YJS. They're completely optional—you can use just YJS, just SQLite, both, or build custom extensions.

### Pure JSON Table Schemas

Table definitions use Standard Schema (ArkType, Zod, etc.) instead of builder functions. This enables:

- **Serialization** for MCP/OpenAPI integration
- **Runtime introspection** for tooling and code generation
- **Type-safe conversions** to validation schemas

## Architecture Overview

Epicenter follows a three-layer data flow:

```
┌────────────────────────────────────────────────────────────────────┐
│  WRITE FLOW                                                         │
│                                                                     │
│  Action Called → Y.Doc Updated → Auto-sync to Extensions           │
│                       │                                             │
│              ┌────────┼────────┐                                    │
│              ▼        ▼        ▼                                    │
│         IndexedDB  SQLite   Markdown                                │
│         (or .yjs)   (.db)   (files)                                 │
└────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────┐
│  READ FLOW                                                          │
│                                                                     │
│  Query Called → Read from Extension (SQLite for complex queries)   │
│                 or directly from Y.Doc (simple lookups)             │
└────────────────────────────────────────────────────────────────────┘
```

## Use Cases

Epicenter is perfect for:

- **Collaborative editors** - Build Google Docs-style applications
- **Local-first apps** - Applications that work offline and sync when ready
- **Desktop applications** - Tauri/Electron apps with embedded databases
- **AI agent tools** - Expose workspaces via MCP for AI assistant integration
- **Multi-device workflows** - Sync data across phone, laptop, and desktop

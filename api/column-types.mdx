## Overview

Epicenter uses [arktype](https://arktype.io) for schema definition. All table rows and KV values are validated against arktype schemas at runtime.

## Imports

```typescript
import { type } from 'arktype';
import { defineTable, defineKv } from '@epicenter/workspace';
```

## Table Schemas

All table schemas must include an `_v` field for versioning:

```typescript
const users = defineTable(
  type({ id: 'string', email: 'string', _v: '1' })
);
```

### Required Fields

Every table row must have:
- `id: 'string'` - Unique row identifier
- `_v: number` - Schema version (literal type like `'1'`, `'2'`, etc.)

### Basic Types

```typescript
// String fields
const users = defineTable(
  type({ id: 'string', name: 'string', email: 'string', _v: '1' })
);

// Number fields
const posts = defineTable(
  type({ id: 'string', views: 'number', likes: 'number', _v: '1' })
);

// Boolean fields
const todos = defineTable(
  type({ id: 'string', completed: 'boolean', _v: '1' })
);
```

### Optional Fields

Mark fields as optional with the `?` suffix:

```typescript
const posts = defineTable(
  type({ 
    id: 'string', 
    title: 'string',
    'description?': 'string',  // Optional
    _v: '1'
  })
);
```

### Array Fields

```typescript
const notes = defineTable(
  type({ 
    id: 'string',
    tags: 'string[]',           // Array of strings
    scores: 'number[]',         // Array of numbers
    _v: '1'
  })
);
```

### Literal Types (Enums)

Use string literal unions for enum-like fields:

```typescript
const tasks = defineTable(
  type({ 
    id: 'string',
    status: "'todo' | 'in_progress' | 'done'",
    priority: "'low' | 'medium' | 'high'",
    _v: '1'
  })
);
```

### Nested Objects

```typescript
const profiles = defineTable(
  type({ 
    id: 'string',
    name: 'string',
    address: {
      street: 'string',
      city: 'string',
      'zipCode?': 'string'
    },
    _v: '1'
  })
);
```

### Nullable Types

```typescript
const posts = defineTable(
  type({ 
    id: 'string',
    title: 'string',
    author: 'string | null',
    _v: '1'
  })
);
```

### Timestamps

Store timestamps as numbers (milliseconds since epoch):

```typescript
const events = defineTable(
  type({ 
    id: 'string',
    createdAt: 'number',
    updatedAt: 'number',
    _v: '1'
  })
);

// Usage
tables.events.set({
  id: '1',
  createdAt: Date.now(),
  updatedAt: Date.now(),
  _v: 1
});
```

### JSON Fields

Use `'unknown'` for fields that can contain any JSON-serializable value:

```typescript
const documents = defineTable(
  type({ 
    id: 'string',
    metadata: 'unknown',
    _v: '1'
  })
);
```

## Schema Versioning

Use the builder pattern for tables with multiple versions:

```typescript
const posts = defineTable()
  .version(type({ id: 'string', title: 'string', _v: '1' }))
  .version(type({ id: 'string', title: 'string', views: 'number', _v: '2' }))
  .migrate((row) => {
    switch (row._v) {
      case 1: return { ...row, views: 0, _v: 2 };
      case 2: return row;
    }
  });
```

### Migration Examples

**Two-version migration:**

```typescript
const posts = defineTable()
  .version(type({ id: 'string', title: 'string', _v: '1' }))
  .version(type({ id: 'string', title: 'string', views: 'number', _v: '2' }))
  .migrate((row) => {
    if (row._v === 1) return { ...row, views: 0, _v: 2 };
    return row;
  });
```

**Three-version migration:**

```typescript
const posts = defineTable()
  .version(type({ id: 'string', title: 'string', _v: '1' }))
  .version(type({ id: 'string', title: 'string', views: 'number', _v: '2' }))
  .version(type({ 
    id: 'string', 
    title: 'string', 
    views: 'number',
    author: 'string | null',
    _v: '3' 
  }))
  .migrate((row) => {
    if (row._v === 1) {
      return { ...row, views: 0, author: null, _v: 3 };
    }
    if (row._v === 2) {
      return { ...row, author: null, _v: 3 };
    }
    return row;
  });
```

## KV Schemas

KV stores don't require `id` or `_v` fields:

```typescript
import { defineKv } from '@epicenter/workspace';

const theme = defineKv(
  type({ mode: "'light' | 'dark'" })
);

const sidebar = defineKv(
  type({ collapsed: 'boolean', width: 'number' })
);
```

### Nested KV Objects

```typescript
const preferences = defineKv(
  type({ 
    editor: {
      tabSize: 'number',
      lineNumbers: 'boolean',
      wordWrap: 'boolean'
    },
    ui: {
      sidebarWidth: 'number',
      showMinimap: 'boolean'
    }
  })
);
```

## Common Patterns

### User Table

```typescript
const users = defineTable(
  type({ 
    id: 'string',
    email: 'string',
    name: 'string',
    'avatar?': 'string',
    createdAt: 'number',
    _v: '1'
  })
);
```

### Post/Article Table

```typescript
const posts = defineTable(
  type({ 
    id: 'string',
    title: 'string',
    content: 'string',
    authorId: 'string',
    published: 'boolean',
    tags: 'string[]',
    createdAt: 'number',
    updatedAt: 'number',
    _v: '1'
  })
);
```

### Task Table with Status

```typescript
const tasks = defineTable(
  type({ 
    id: 'string',
    title: 'string',
    status: "'todo' | 'in_progress' | 'done'",
    priority: "'low' | 'medium' | 'high'",
    'dueDate?': 'number',
    _v: '1'
  })
);
```

### Settings KV

```typescript
const settings = defineKv(
  type({ 
    theme: "'light' | 'dark' | 'system'",
    language: 'string',
    notifications: 'boolean',
    autoSave: 'boolean'
  })
);
```

## Type Inference

TypeScript automatically infers types from your schemas:

```typescript
const users = defineTable(
  type({ id: 'string', email: 'string', name: 'string', _v: '1' })
);

const workspace = defineWorkspace({ id: 'app', tables: { users } });
const client = createWorkspace(workspace);

// TypeScript knows the exact type:
// { id: string; email: string; name: string; _v: 1 }
client.tables.users.set({
  id: '1',
  email: 'user@example.com',
  name: 'John Doe',
  _v: 1  // Note: runtime value is number, not string
});
```

## Advanced arktype Features

For more advanced validation, refer to the [arktype documentation](https://arktype.io):

- Email validation: `'email'`
- URL validation: `'string.url'`
- Numeric constraints: `'number>=0'`, `'number>0'`
- String parsing: `'string.numeric.parse'`
- Custom constraints and transformations

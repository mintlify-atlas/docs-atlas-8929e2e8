## Overview

The `sqliteProvider()` function creates a SQLite extension that syncs workspace data to a SQLite database and provides SQL query capabilities via Drizzle ORM.

```typescript
import { sqliteProvider } from '@epicenter/workspace';
```

## Storage Locations

The SQLite provider stores files in the following locations:

- **Database**: `.epicenter/providers/sqlite/{workspaceId}.db`
- **Logs**: `.epicenter/providers/sqlite/logs/{workspaceId}.log`

## Basic Usage

### Setup with createWorkspace

```typescript
import { createWorkspace, sqliteProvider } from '@epicenter/workspace';

const workspace = createWorkspace(definition)
  .withExtension('sqlite', (c) => sqliteProvider(c));
```

### Query with Drizzle ORM

```typescript
import { eq, desc, count, avg } from '@epicenter/workspace';

const blogActions = {
  getPublishedPosts: defineQuery({
    handler: async () => {
      const { sqlite } = workspace.extensions;
      return await sqlite.posts
        .select()
        .where(eq(sqlite.posts.published, true))
        .orderBy(desc(sqlite.posts.publishedAt))
        .limit(10);
    },
  }),

  getPostStats: defineQuery({
    handler: async () => {
      const { sqlite } = workspace.extensions;
      return await sqlite.posts
        .select({
          category: sqlite.posts.category,
          total: count(),
          avgViews: avg(sqlite.posts.views),
        })
        .groupBy(sqlite.posts.category);
    },
  }),
};
```

## Advanced Features

### Manual Sync Operations

You can manually trigger sync operations between YJS and SQLite:

```typescript
const blogActions = {
  syncToSqlite: defineMutation({
    handler: () => workspace.extensions.sqlite.pullToSqlite(),
  }),
  
  syncFromSqlite: defineMutation({
    handler: () => workspace.extensions.sqlite.pushFromSqlite(),
  }),
};
```

### Direct Database Access

Access the raw Drizzle database instance for custom queries:

```typescript
const { db } = workspace.extensions.sqlite;

// Raw SQL query
const result = await db.execute('SELECT * FROM posts WHERE published = true');
```

## How It Works

1. **Automatic Sync**: Observes YJS changes and updates SQLite automatically
2. **WAL Mode**: Uses Write-Ahead Logging for concurrent access
3. **Loop Prevention**: Prevents infinite loops with sync coordination flags
4. **Error Logging**: Logs validation errors without blocking sync
5. **Initial Sync**: Performs full sync on startup to ensure consistency
6. **Schema Mapping**: Automatically creates SQLite tables matching your workspace schema

## Type Safety

The SQLite provider is fully typed based on your workspace definition:

```typescript
import { type } from 'arktype';

const definition = defineWorkspace({
  id: 'blog',
  tables: {
    posts: defineTable(
      type({
        id: 'string',
        title: 'string',
        published: 'boolean',
        _v: '1',
      })
    ),
  },
});

const workspace = createWorkspace(definition)
  .withExtension('sqlite', sqliteProvider);

// ✅ Fully typed - editor autocomplete works
workspace.extensions.sqlite.posts.select();

// ❌ Type error - 'articles' doesn't exist
workspace.extensions.sqlite.articles.select();
```

## Overview

The `markdownProvider()` function creates a markdown extension that persists workspace data as human-readable markdown files with YAML frontmatter.

```typescript
import { markdownProvider } from '@epicenter/workspace';
```

## Storage Locations

The markdown provider stores files in the following locations:

- **Markdown files**: `./{workspaceId}/{tableName}/*.md` (configurable)
- **Logs**: `.epicenter/providers/markdown/logs/{workspaceId}.log`
- **Diagnostics**: `.epicenter/providers/markdown/diagnostics/{workspaceId}.json`

## Basic Usage

### Default Configuration

```typescript
import { createWorkspace, markdownProvider } from '@epicenter/workspace';

const workspace = createWorkspace(definition)
  .withExtension('markdown', (c) => markdownProvider(c));
```

With default configuration:
- **Frontmatter**: All columns except `content`
- **Body**: `content` column (if exists)
- **Filename**: `{id}.md`

### Custom Directory

```typescript
const workspace = createWorkspace(definition)
  .withExtension('markdown', (c) =>
    markdownProvider(c, {
      directory: './data',
    })
  );
```

### Custom Serialization

```typescript
import { basename } from 'node:path';
import { Ok, Err } from '@epicenter/workspace';

const workspace = createWorkspace(definition)
  .withExtension('markdown', (c) =>
    markdownProvider(c, {
      tableConfigs: {
        posts: {
          directory: './posts',
          serialize: ({ row }) => ({
            frontmatter: {
              title: row.title,
              published: row.published,
              date: row.publishedAt,
            },
            body: row.content,
            filename: `${row.id}.md`,
          }),
          deserialize: ({ frontmatter, body, filename }) => {
            const id = basename(filename, '.md');
            return Ok({
              id,
              content: body,
              title: frontmatter.title,
              published: frontmatter.published,
              publishedAt: frontmatter.date,
            });
          },
        },
      },
    })
  );
```

const blogActions = {

## How It Works

1. **File Watching**: Watches markdown directories for file changes
2. **Bidirectional Sync**: Syncs changes between markdown files and YJS
3. **Mapping**: Maintains rowId ↔ filename mapping (handles renames/deletions)
4. **Validation**: Validates all files on startup
5. **Error Tracking**: Tracks errors in diagnostics (JSON) and error log (append-only)
6. **Loop Prevention**: Prevents infinite loops with sync coordination flags

## Error Handling

### Diagnostics File

Validation errors are tracked in a JSON diagnostics file:

```json
{
  "posts/invalid.md": {
    "error": "Missing required field: title",
    "timestamp": "2024-03-15T10:30:00Z"
  }
}
```

### Error Log

An append-only log file records all errors:

```
[2024-03-15T10:30:00Z] posts/invalid.md: Missing required field: title
```

## Advanced Examples

### Custom Filename Generation

```typescript
serialize: ({ row }) => ({
  frontmatter: { title: row.title },
  body: row.content,
  filename: `${row.publishedAt}-${row.slug}.md`,
}),
```

### Nested Frontmatter

```typescript
serialize: ({ row }) => ({
  frontmatter: {
    meta: {
      title: row.title,
      author: row.author,
    },
    seo: {
      description: row.description,
      keywords: row.keywords,
    },
  },
  body: row.content,
  filename: `${row.id}.md`,
}),
```

### Error Handling in Deserialize

```typescript
deserialize: ({ frontmatter, body, filename }) => {
  if (!frontmatter.title) {
    return Err(
      MarkdownProviderErr({
        message: 'Missing required field: title',
        context: { filename },
      })
    );
  }
  
  return Ok({
    id: basename(filename, '.md'),
    title: frontmatter.title,
    content: body,
  });
},
```

## Type Safety

The markdown provider validates data against your workspace schema:

```typescript
const definition = defineWorkspace({
  id: 'blog',
  tables: {
    posts: {
      id: id(),
      title: text(),
      content: text({ nullable: true }),
      published: boolean({ default: false }),
    },
  },
});

const workspace = createWorkspace(definition)
  .withExtension('markdown', markdownProvider);

// Validation errors are logged but don't block sync
// Invalid files are tracked in diagnostics
```
